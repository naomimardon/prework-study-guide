HTML

  • Always start with !DOCTYPE html
  • Then html lang="en" (if building in English)
  • The head element contains information about the webpage. This includes metadata and the title of the webpage
  • The body element represents the visible content shown to the user
  • The header element contains the title of the webpage
  • The main element contains the main content of the webpage
  • The section elements are semantic elements that define a section in a document that contains thematic content
  • h1, h2, h3 etc represent headings
  • ul represents an unordered (or bullet point) list
  • ol represents an ordered (or numbered) list
  • li represents a new line
  • p represents a paragraph or block of text
  • img represents an image; the src attribute links to the location of the image; alt provides a text description of the image
  • br creates a line break, often used after images
  • a stands for anchor to create links to the same or other webpages. The href attribute points to the URL for the link

CSS

  • margin: indicates how much space we want around the outside of an element
  • padding: indicates how much space we want around the content inside an element
  • text align: left, center or right
  • display:block assigns the element to start on a new line
  • border: assigns a border
  • box-shadow: adds shadow effects
  • colours can be assigned using semantic reference or hex #

  • period preceding name of class (e.g. card): designates a class sector so any HTML element assigned to the class card will gain these property assignments from the CSS
  • link the CSS to the HTML: rel specifies the relationship between the current doc and the linked doc resource; href specifies the location (URL) of the external resource. style
  • css is linked in the head element

Git

  • git status: checks what branch you are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout main: takes you back to the main branch
  • git pull origin main: pulls down the latest main branch
  • git add -A: adds a feature to the working branch
  • git commit -m "commit message": stages files to be committed, with a message describing what the commit contains
  • git push origin branch-name: pushes changes to the remote branch

JavaScript

  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • console.log outputs a message to the file
  • An array is a single variable that is used to hold a group of data
  • An array index identifies the location of each data item in the array with a number, starting at 0
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over
  • A conditional if statement can be used to interrupt the control flow by executing a block of code a specific number of times
  • A function is a set of instructions that tells the computer how to perform a certain task
  • Remember to include empty parentheses after a function name
  • To call a function, add its name in the JS file in the position you want the code to execute
  • Math.floor() and Math.random() functions can be used together to randomise a selection
  • script the JS to the HTML: src specifies the location
  • script.js is linked within the body element
